This week's workshop introduced members to Git and GitHub for version control.
Whether you're writing code, managing data analysis scripts, or collaborating
on a group project, version control keeps your work organized and recoverable.

## What We Covered

- Initializing a repository and making commits
- Branching and merging workflows
- Collaborating through pull requests on GitHub
- Resolving merge conflicts

Attendees practiced by creating their own repositories and submitting pull
requests to a shared demo project. By the end of the session, everyone had a
working understanding of the basic Git workflow.

## Next Steps

If you want to keep practicing:

1. Create a personal GitHub repository for a project you're working on
2. Try the [Git Novice lesson](https://swcarpentry.github.io/git-novice/)
   from Software Carpentry
3. Contribute to an open-source project during our upcoming Hacktoberfest event
